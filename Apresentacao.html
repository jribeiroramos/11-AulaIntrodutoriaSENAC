Mas... o que é um banco de dados?

Quando vocês ouvem Banco de Dados, o que vem à mente?

  • Exemplos: planilhas, sistemas de notas, cadastros, etc.
  • Quantos bancos de dados vocês conhecem?

Banco de dados é uma coleção organizada de dados que:

  • Facilita armazenamento, busca e atualização.
  • É manipulado por linguagens como SQL:
    • SELECT para consultar dados.
    • INSERT para adicionar informações.
    • UPDATE para modificar dados existentes.
    • DELETE para remover dados.

Exemplo SQL:

SELECT nome, idade FROM clientes WHERE idade > 18;

Dado x Informação

O que é um dado? Um fato bruto, sem contexto ou interpretação.

O que é informação? Um dado interpretado e contextualizado para criar significado.

  • Exemplo: Dado: "25" → Informação: "25 anos" (idade de uma pessoa).

Modelando um Banco de Dados

Passos principais:

  • Entidade: Objeto ou conceito do mundo real que você deseja armazenar dados. Pode ser qualquer coisa sobre a qual você precise guardar informações no seu sistema.
  • Identificar as entidades (e.g., clientes, produtos).
  • Atributo é uma característica ou propriedade de uma entidade.
  • Definir os atributos de cada entidade (e.g., nome, data de nascimento).
  • Relacionamento define como duas ou mais entidades se conectam dentro do banco de dados. Ele estabelece uma ligação entre os dados de diferentes entidades, permitindo que as informações sejam organizadas e consultadas de forma eficiente.
  • Estabelecer relacionamentos entre as entidades (e.g., clientes compram produtos).

Dados bem modelados x Dados mal modelados

Dados bem modelados:

  • Facilitam consultas e análises.
  • Reduzem redundâncias.
  • Garantem integridade e consistência.

Dados mal modelados:

  • Dificultam análises.
  • Causam inconsistências.
  • Aumentam o esforço de manutenção.
